Scope

What SIGIL inspects today.

The current implementation is intentionally narrow, but the pieces are connected around the one thing that matters: local AI runtime safety.

Ollama model stores

Walks Ollama manifests and blobs, verifies SHA-256, parses registry / namespace / model / tag provenance, flags missing or invalid artefact references.

License detection

Reads the application/vnd.ollama.image.license layer and identifies SPDX shortnames plus full-body matches for ten licenses: Apache-2.0, MPL-2.0, the GPL / LGPL family, BSD-2 / 3-Clause, ISC, and MIT.

Runtime exposure

Classifies the configured Ollama host and inspects local listening sockets via /proc: localhost, LAN, public bind, docker-published, or reverse proxy. No subprocesses spawned.

Native execution surfaces

Loads x86_64 objects, lifts a small instruction subset into IR and SafeISA, maps external symbols such as network and process calls to capability evidence.

One report

Verdict beside the evidence.

SIGIL renders the policy verdict next to the call site, the capability table, and the lifted SafeISA — so a reviewer can read the verdict and the reason together.

  • Verdict banner: [PASS] / [WARN] / [FAIL]
  • Capability table keyed by address and instruction
  • Policy violations carry the call site and the symbol
  • SafeISA fenced block shows blocked external calls as CALL_STUB

AI-BOM

Provenance and license, in one file.

aibom generate --runtime ollama emits the same schema-1.1 JSON / Markdown for every local model — runtime exposure, provenance tuple, detected SPDX id, and a layer table keyed by digest.

  • Stable schema_version: "1.1" JSON contract
  • Runtime property table and observed binds
  • License card with SPDX id and digest
  • Provenance: registry / namespace / model / tag
  • Findings table grouped by severity and category

Difference

Not a model catalog.

SIGIL treats a local LLM deployment as an environment: model artefacts, runtime exposure, native binaries, and policy all matter, together.

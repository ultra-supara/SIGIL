What SIGIL inspects today

The current implementation is intentionally narrow, but the pieces are already connected around local AI runtime safety.

Ollama model stores

Walks Ollama manifests and blobs, verifies SHA-256, parses registry / namespace / model / tag provenance, and flags missing or invalid artifact references.

License detection

Reads the application/vnd.ollama.image.license layer and identifies SPDX shortnames plus full-body matches for ten licenses, including Apache-2.0, MPL-2.0, the GPL/LGPL family, BSD-2/3-Clause, ISC, and MIT.

Runtime exposure

Classifies the configured Ollama host and inspects local listening sockets via /proc — localhost, LAN, public bind, docker-published, or reverse proxy. No subprocesses spawned.

Native execution surfaces

Loads x86_64 objects, lifts a small instruction subset into IR and SafeISA, and maps external symbols such as network and process calls to capabilities.

One report, evidence in line

SIGIL renders the policy verdict next to the call site, the capability table, and the lifted SafeISA — so a reviewer can read the verdict and the reason together.

  • Verdict banner: [PASS] / [WARN] / [FAIL]
  • Capability table keyed by address and instruction
  • Policy violations carry the call site and the symbol
  • SafeISA fenced block shows blocked external calls as CALL_STUB

Why this is different from a model catalog

SIGIL treats a local LLM deployment as an environment: model artifacts, runtime exposure, native binaries, and policy all matter.

Inspection axis SIGIL direction Current status
Model artifact identity Manifest + blob SHA-256 verification Implemented for Ollama
Provenance registry / namespace / model / tag tuple Implemented for Ollama
License metadata SPDX id from shortname or body match 10 licenses detected (Apache-2.0, MIT, MPL-2.0, GPL/LGPL, BSD-2/3, ISC)
Runtime exposure API endpoint + /proc bind classification Implemented (Linux /proc, degrades elsewhere)
Native capability evidence External calls + guarded SafeISA lifting Narrow x86_64 path
